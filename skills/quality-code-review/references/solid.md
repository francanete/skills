# SOLID Violations

Flag any violations of:

- **Single Responsibility** — a class/module should have one reason to change.
- **Open/Closed** — open for extension, closed for modification.
- **Liskov Substitution** — subtypes must be usable wherever the base type is expected, without breaking behaviour.
- **Interface Segregation** — many small focused interfaces beat one fat interface.
- **Dependency Inversion** — depend on abstractions, not concretions. High-level modules should not depend on low-level details.
