# Comments and Formatting

## Comments

- Flag redundant comments that restate the code.
- Flag misleading or outdated comments.
- Flag commented-out code (version control exists).
- Acceptable: legal headers, *why* explanations, TODOs for known debt, consequence warnings.

## Formatting

- Blank lines between concepts, related lines grouped together.
- Variables declared close to usage.
- Dependent functions vertically close, caller above callee.
- Lines within 80–120 characters.
