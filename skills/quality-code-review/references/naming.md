# Naming

- Names are intention-revealing — you can understand *why* it exists and *what* it does without a comment.
- No disinformation — names don't hint at something different from what the code does.
- Meaningful distinctions — no noise words (`data`, `info`, `manager`) unless they genuinely clarify.
- Pronounceable and searchable — no cryptic abbreviations.
- No type encoding in names (no Hungarian notation, no `strName`).
