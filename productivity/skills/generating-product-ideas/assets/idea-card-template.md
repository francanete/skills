# Idea Card Template

Use this exact structure for each validated idea, in ranked order (highest score first).

```
### Idea N: <short, specific, benefit-oriented title>

**Product change** — <1–2 sentences on what the user actually experiences>
**Commercial angle** — <1 sentence: conversion / retention / ARPU / activation / expansion / virality / referrals>
**Architectural fit** — <1 sentence on how it slots into the existing system>
**Market position** — novel / table-stakes-but-missing / already-shipped-by-<competitor>
**Tech difficulty** — S / M / L (~hours / days / weeks)
**Recommended approach** — <2–3 sentences, high-level only. No file paths unless genuinely critical.>
```

If the validator revised this idea, append one line below the block:

```
*Revised after validation: <one-line summary of what changed and why>*
```

## Closing block (after all ideas)

In this order, omitting any section that has no content:

```
### Cross-cutting concerns
<bulleted list from validator findings>

### Dropped by validator
- <Title> — <validator reason>
- <Title> — <validator reason>

### Suggested ordering
<one sentence — only if there's a clear winner>

### Noted separately
<single line — bugs spotted in discovery, not counted as ideas>

---

Want me to plan any of these in depth?
```

The final question is verbatim. Do not paraphrase.
